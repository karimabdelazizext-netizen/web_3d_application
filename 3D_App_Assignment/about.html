About This 3D Web Application

Design

I, Karim Ayman, declare that this 3D Web Application is my own work. I have utilized the Three.js library and Bootstrap 5 framework to build the web application. All 3D models were created by me in Blender 4.0, specifically designed as a 'Perfume Collection' to demonstrate geometric variety (Cube, Cylinder, and Sphere primitives) while maintaining a consistent brand identity.

Integration of 3D Models

I developed a premium perfume line. I utilized Shade Smooth and Subdivision Surface modifiers to ensure a high-end glass appearance, while keeping the poly-count low for efficient web performance.

Interaction Design

The app follows a the MVC pattern. I created a dynamic loadModel function that removes any old geometries and materials from the GPU memory before loading new ones to prevent memory leaks.

Implementation Details

I integrated OrbitControls to allow users to interact with my products from any angle. I also included a Wireframe Toggle to show the underlying geometry, satisfying the 'Content Swapping' requirement.

Implementation and Publication

The application uses a Fluid Grid Layout (Bootstrap 5) to ensure it is fully responsive across mobile, tablet, and desktop views.

High-contrast UI elements and semantic HTML tags were used to improve readability and screen-reader compatibility.

Deeper understanding

UV Mapping and Texture Mapping: UV coordinates are a 2D coordinate system used to map 2D textures onto 3D geometry. Unlike 3D XYZ coordinates, UV represents 2D texture space (U = horizontal, V = vertical). This application dynamically generates bottle labels using HTML Canvas and applies them to bottle meshes via UV mapping. Each bottle model is assumed to have pre-existing UV coordinates from the Blender export; the texture loader checks for these coordinates and applies a custom-generated label texture (created with product name, branding, and gradient backgrounds) to the color map of the material.

Material Properties in Three.js: I used MeshStandardMaterial which supports physically-based rendering. The textures are combined with metalness (0.2) and roughness (0.1) properties to create a premium glass appearance. The transparency property is set to true with opacity at 0.95, allowing for subtle light refraction effects that make the bottles appear more realistic.

Canvas Texture Generation: Rather than pre-rendering texture files, my application generates labels dynamically using the HTML5 Canvas API. This approach offers several advantages: reduced file size (no separate image assets), dynamic text rendering (product names can be modified at runtime), and seamless integration with Three.js's CanvasTexture class. The canvas texture is rendered at 512x512 resolution with proper filtering to maintain quality when viewed from various angles.

Memory Management: The model loading process includes proper resource disposal. Before loading a new model, the old model's geometry and materials are removed from GPU memory using Three.js's dispose() method. This prevents memory leaks common in WebGL applications where textures and geometries can accumulate if not properly cleaned up.

Advanced Interactive Features

Texture Mode Switching: my application implements three different texture generation modes accessible via UI buttons. The 'Original' mode displays standard gradient labels with branding text. The 'Holographic' mode creates diagonal gradient effects with metallic yellow accents and "LIMITED EDITION" branding. The 'Minimalist' mode generates clean white backgrounds with serif typography and subtle borders. Each mode is dynamically generated via Canvas, allowing instant switching without pre-loaded assets.

Real-Time Color Customization: Users can select any color using the HTML5 color picker, and the application recalculates and reapplies the texture in real-time. The color update maintains the current texture mode (Original, Holographic, or Minimalist), updating only the material's texture map without requiring geometry re-computation.

Camera Preset System: The camera system uses eased animation to smoothly transition between preset viewpoints (Front, Side, Top, Isometric). This is implemented using requestAnimationFrame and a quadratic easing function for smooth, professional transitions. Preset positions are hard-coded camera vectors that provide optimal viewing angles for product inspection.

Scene Lighting Architecture: The scene uses a multi-light setup: 1) Ambient Light (0xffffff, intensity 1.5) for overall scene illumination, 2) Directional Light (0xffffff, intensity 1.2) from upper-right for main shading, 3) Gold Fill Light (0xc9a84c) positioned below for warm rim lighting, and 4) Blue Rim Light (0x4488ff) from behind for atmospheric depth. This setup creates a professional studio environment similar to high-end product photography.

OrbitControls Implementation: The application integrates Three.js OrbitControls to allow users to rotate, zoom, and pan around models. Damping is enabled (dampingFactor: 0.05) for smooth, inertia-based interaction. This control system works seamlessly with the camera preset system—presets set the camera position while OrbitControls manage user interaction.
